Описание проекта

Telegram-бот для планирования задач и управления напоминаниями, написанный на Python с использованием библиотеки aiogram 3.x.

Бот позволяет пользователям создавать события с указанием даты и времени, выбирать тип напоминания (один раз, каждый день, каждую неделю, каждый месяц, по умолчанию) и получать автоматические уведомления в нужный момент.

Администратор может просматривать все события пользователей (опционально).

Все данные хранятся в локальной базе данных SQLite (db.db).

---

Основные функции

Для пользователя:

· ✅ Создание событий с названием, описанием и датой
· 🔔 Выбор типа напоминания:
  · Один раз – напоминание ровно в момент события
  · Каждый день – ежедневные напоминания
  · Каждую неделю – еженедельные напоминания
  · Каждый месяц – ежемесячные напоминания
  · По умолчанию – напоминания за день и за час
· 📋 Просмотр списка моих событий
· ⏰ Автоматические уведомления в нужное время
· 💾 Все события сохраняются после перезапуска бота

Для администратора (в планах):

· Просмотр всех событий пользователей
· Рассылка уведомлений

Таблицы баз данных

Таблица users

Поле Тип Описание
id INTEGER ID пользователя Telegram (PRIMARY KEY)
status INTEGER Текущий статус диалога (1-5)
name TEXT Временное название события
comment TEXT Временный комментарий
event_time TEXT Временное время события

Таблица events

Поле Тип Описание
event_id INTEGER ID события (PRIMARY KEY)
id INTEGER ID пользователя (FOREIGN KEY)
name TEXT Название события
comment TEXT Описание события
time TEXT Дата и время события в формате ДД.ММ.ГГГГ ЧЧ:ММ
current_time TEXT Время создания события

Таблица reminders

Поле Тип Описание
id INTEGER ID напоминания (PRIMARY KEY)
user_id INTEGER ID пользователя
event_id INTEGER ID события
title TEXT Название события
description TEXT Описание события
event_time TEXT Время события
remind_at TEXT Когда отправить напоминание
is_sent INTEGER Статус отправки (0 = не отправлено, 1 = отправлено)
reminder_type TEXT Тип напоминания (once/daily/weekly/monthly/default)

Инструкция по запуску

Windows / macOS / Linux

1. Клонирование проекта

```bash
git clone https://github.com/yourusername/telegram-planner.git
cd telegram-planner
```

2. Создание и активация виртуального окружения

Windows (PowerShell):

```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
```

Если PowerShell выдает ошибку выполнения скриптов, выполните один раз:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

Windows (CMD):

```cmd
python -m venv venv
venv\Scripts\activate.bat
```

macOS / Linux:

```bash
python3 -m venv venv
source venv/bin/activate
```

При успешной активации в начале строки терминала появится префикс (venv).

3. Установка зависимостей

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

4. Настройка токена

Создай файл config.py в корне проекта:

```python
TOKEN = "ваш_токен_от_BotFather"
```

Получить токен можно у @BotFather в Telegram.

5. Запуск бота

```bash
python main.py
```

Бот запустится в режиме длительного опроса (long polling). Для остановки нажмите Ctrl+C.

Возможные проблемы и решения

1. Ошибка ModuleNotFoundError: No module named 'aiogram'

Решение: Установи зависимости заново:

```bash
pip install -r requirements.txt
```

2. Ошибка при активации виртуального окружения в PowerShell

Решение: Выполни команду:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

3. Бот не отвечает

Проверь:

· Правильно ли указан токен в config.py
· Есть ли интернет-соединение
· Не заблокирован ли бот пользователем

4. Напоминания не отправляются

Проверь:

· Запущен ли планировщик (он запускается автоматически с ботом)
· Правильно ли сохранена дата в БД
· Не перезапускался ли бот (данные сохраняются, всё восстановится)
